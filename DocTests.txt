1. Type Mapping & Binding (Row, RowMetadata, Parameters)

Focuses on io.r2dbc.spi.Row consumption, native TDS-to-Java type conversion, and verifying driver metadata compliance.

	TdsClientAllDataTypes
	TdsClientExhaustiveNumeric
	TdsClientExhaustiveNonNumeric
	TdsClientBindingMatrixSymmetry
	TdsClientTypeMatrix
	TdsClientXmlStream

2. Large Object Handling (Blob, Clob)

Focuses on reactive stream compliance for large data chunks (io.r2dbc.spi.Blob and Clob) to ensure memory and stream boundaries are maintained without exhausting the heap.

	TdsClientLob
	TdsClientLobBinding
	TdsClientLobOk
	TdsClientLobBug

3. Parameterized Statement Execution (Statement)

Focuses on io.r2dbc.spi.Statement interactions. This includes standard query execution, parsing Out Parameters (Result.OutSegment), and parameterized batching via the Statement.add() API.

	TdsClientTestSelect
	TdsClientSelectOne
	TdsClientTestOutParams
	TdsClientBatchLobBinding *(Validating stream behavior when parameterized batching is mixed with LOBs)*
	TdsClientEssential
	TdsClientTestAll

4. Static Batch Execution (Batch)

Strictly isolated to the io.r2dbc.spi.Batch interface for executing multiple, parameter-less raw SQL strings in a single network request.

	TdsClientBatch
	TdsClientBatchChaos
	TdsClientSqlBatchChaos

5. Transaction Management (Connection)

Focuses on the lifecycle of transactions (beginTransaction, commitTransaction, rollbackTransaction) and isolation level configurations.

	TdsClientTransactionSimple

6. Error Translation & Diagnostics (R2dbcException)

Ensures native database errors and corrupted packets are gracefully caught and correctly mapped into the standard io.r2dbc.spi.R2dbcException hierarchy.

	TdsClientExceptions
	TdsClientError
	TdsClientMisbehavior
	TdsClientMicroPoison
	TdsClientEomParadox

7. Reactive Streams Compliance & Concurrency

Validates that the client strictly obeys Reactive Streams specifications (backpressure, thread safety, execution ordering) across heavy asynchronous boundaries.

	TdsClientOrder
	TdsClientOrderSync
	TdsClientOrderSyncLob
	TdsClientRandom
	TdsClientRandomAsync
	TdsClientRandomPool
	TdsClientChaos
	TdsClientMacroChaos
	TdsClientLobChaos
	TdsClientSpStress

8. Observability & Extensions

Tests features that observe or intercept the core SPI, typically via connection factories or wrappers.

	TdsClientFilter
	TdsClientTraceId
